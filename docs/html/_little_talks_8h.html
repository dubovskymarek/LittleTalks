BOOL LT_ForceSubscribeRemoteTopic(LT_DEVICE_ID deviceId, LT_TOPIC_ID topicId)

It's special advanced function for immediately connect remote device and subscribe remote topic. This function use only situation, when you cannot wait for standard connection (10-20 seconds), but you need send topic in few miliseconds. For example: Android device is in sleep mode, every communication is stoped, and sometimes every 5 minutes i call small function. Android cannot wake up but it's allow me call small function few miliseconds, but every devices are disconnected, and they need 10-20 seconds to reconnect (this time it's not supported by sleeping android). To resolve this problem, is ignore standard connection process and force this process at one moment.

example using: LT_ForceSubscribeRemoteTopic(remoteDeviceId, deviceIp, topicId); LT_Publish(topicId, valuePtr, valueSize);

Parameters
deviceIdremote device id
topicIdremote topic
Returns
Return if force subscribe is success then return true, otwerwise return false
